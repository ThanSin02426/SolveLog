# SolveLog Privacy Policy

**Effective date:** 24 June 2026

SolveLog is a browser extension that saves a user's accepted coding-problem submissions to a location chosen by the user.

## Data the extension processes

SolveLog processes the code submitted by the user, the problem title and identifier, public topic tags, difficulty, runtime, memory usage, submission identifier, and the configured GitHub repository details.

When GitHub mode is enabled, SolveLog also stores the GitHub token entered by the user in the browser's local extension storage.

## How data is used

The data is used only to generate solution files and either:

- commit those files to the GitHub repository selected by the user; or
- download those files to the user's computer.

## Data sharing

SolveLog sends data only to services needed for its single purpose:

- LeetCode, to read the current problem's public metadata and observe the user's own submission result;
- GitHub API, when the user enables GitHub mode, to write generated files to the selected repository.

SolveLog does not sell data, use it for advertising, create behavioural profiles, or transfer it to data brokers.

## Analytics

The extension does not include analytics, tracking pixels, advertising SDKs, or remote executable code.

## Storage and retention

Settings, retry items, recently processed submission identifiers, and the GitHub token are stored locally in the user's browser profile. The user can remove the token from SolveLog settings or remove all local extension data by uninstalling the extension.

Generated files remain in the location chosen by the user until the user deletes them.

## Security

Users are instructed to create a fine-grained GitHub token limited to one repository with only Contents read/write permission. The token is not synchronised through the user's Google account.
